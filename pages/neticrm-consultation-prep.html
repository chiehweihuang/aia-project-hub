一頁總結

三個核心答案
  1. 「中英雙語建置加購 NT$10,500」是什麼?:netiCRM 預設只給繁體中文募款頁,此一次性加購讓他們替你建一個英文版頁面(英中切換或兩個獨立網址),用於 ADALS 國際與會者 / 海外贊助。AIA 目前可能不需要立刻買 — 看諮詢結果(見 §1)。

§1 EXPLAINER

「中英雙語建置加購 NT$10,500」到底是什麼?

這個項目在 payment-stack-decision §2 規格表出現,Recur.tw 那一欄寫「未提」。先解釋兩個欄位各自的意思,再說 AIA 該不該買。

netiCRM 欄:NT$10,500 加購 = 一次性「建一個英文版」服務費

  • 預設狀態:netiCRM 微型方案買來後,所有對外頁面(募款頁、報名頁、收據)都是繁體中文 only。後台管理介面同樣是繁中。
  • NT$10,500 涵蓋什麼(依公開資訊推測,須諮詢確認):netiCRM 工程師替 AIA 建一份「英文版的對外頁面」,讓海外捐款者 / 報名者能用英文流程。技術上是 CiviCRM 多語言模組的啟用 + 內容翻譯協作 + UI 文字英文化。
  • 不涵蓋什麼:後台管理(admin)介面;這部分仍是繁中,給 AIA 工作人員用。
  • 是一次性還是年費?:不確定。「建置加購」這個名稱看似一次性,但若 netiCRM 後續需要維護雙語會不會收年費,要問。

Recur.tw 欄:「未提」= Recur 公開資料沒寫

  • 「未提」不是「沒有」也不是「有」 — 是 Recur 的公開定價 / spec 頁面沒有明確提到是否支援中英雙語。
  • 本週諮詢 Recur 時可以順便問:有沒有英文版表單?跨幣別?雙語頁面客製?
  • 但提醒:即使 Recur 有英文版,還是不解決 §3 提到的 NPO 法規層缺失問題。
AIA 該不該買這 NT$10,500? 三個決定關鍵:
  1. ADALS 2026 海外與會者比例:若 < 10% 海外與會者,可能不需要 — 把 AIA 主站做雙語,只把 netiCRM 表單 label 改為英文也許足夠。若 > 30%,雙語頁明顯加分。
  2. 「英文版」具體範圍:只要英文表單,還是整頁英文(含頁面標題、說明文字、感謝信、收據)?範圍越大越值得加購。
  3. 替代方案存在嗎:能不能用「兩份獨立中文 / 英文表單」,各自設不同 URL,放在 AIA 主站不同語言頁面?如果 netiCRM 支援這種「複製表單 + 改 label」,可能不用花 NT$10,500。問 netiCRM 確認。
§2 CONTEXT

AIA 自建網站現況與整合需求

現況

  • 主站平台:WordPress + Elementor Page Builder(per work-structure.html)
  • 設計風格:有既定 Style Guide(Brand 色彩、字體、Logo)
  • 無障礙:目標 WCAG 2.2 AA(AIA 是 a11y NPO,自家站不能不無障礙)
  • 語言:目標中英雙語(海外 a11y community / 國際 a11y experts)
  • 建置者:Christina 是關鍵路徑(所有頁面)

需要 netiCRM 接的場景

場景規模對 netiCRM 的需求
ADALS 2026 報名 + 收款150-200 人表單(中 + 英)+ 信用卡 / ATM / 超商代碼 + 收據 + 報名管理
海外個人購票USD 60-500 / 人國際卡 + 英文表單 + 英文收據
企業贊助 invoiceNT$50K-300K線上不收錢,只收意向;invoice 線下開

關鍵 framing:netiCRM 不是來「替代 AIA 主站」 — 是來當主站的金流 / 報名 / CRM 後端。AIA 站負責品牌敘事 + 內容呈現;netiCRM 負責「按鈕一按開始繳費 / 開收據 / 進 CRM」這條路徑。
§3 VALUE

netiCRM 對 AIA 的具體價值(vs 全自建)

「為什麼不自己用 WordPress + 5 個 plugin 拼出來?」這個問題的答案就是 netiCRM 的價值所在。下面 7 項是 AIA 自建會做不到或做得很痛苦的事。

1. 台灣 NPO 抵稅收據(architectural,不能後補)

  • 合規收據格式(統編、捐款扣抵編號、社團法人格式)直接內建
  • AIA 取得所得稅法 §11(4) 認定後,netiCRM 後台一個 toggle 就 unlock 抵稅功能
  • 自建要找會計師 + 工程師對接國稅局格式 — 估 1-3 個月工時

2. 公益勸募條例合規欄位 + 年度報表

  • 內政部「公益勸募條例」要求的欄位(勸募字號、用途、執行狀況)內建
  • 年度勸募申報的資料可篩選 / 匯出(完整報表產出能力需向 netiCRM 確認)
  • 自建要研讀條例 + 改表單 + 寫匯出邏輯

3. 統一編號 handling(企業捐款者)

  • 企業捐款者填統編,系統自動帶到收據
  • 統編格式驗證(8 位數字)、查核機制(若有對接)
  • 自建要寫驗證邏輯 + 跟政府 OpenAPI 對接

4. 跨年 donor / 會員 CRM(這是「CRM」的本質)

  • 每位 donor 的捐款歷程、互動紀錄、寄信狀態、會員資格
  • 不限自訂欄位 / 標籤 / 群組
  • 自建只能用 Google Sheets 對單 — 50 人以後變人類負擔

5. Event Registration(ADALS 主幹)

  • 活動報名頁(無限建置)、QR Code 簽到、優惠碼、多票種
  • 報名 + 繳費 + 收據一條龍
  • 自建要 WordPress + Events Calendar Pro + Event Tickets Plus + 收據邏輯 + 串金流 = 5 個 plugin 拼裝(maintainability 失敗,見 payment-stack §7.6)

6. 已串好的金流(藍新 / MyPay / 國際卡)

  • 不用自己接金流 SDK
  • 金流出問題 netiCRM 客服可協助 escalation
  • 自建要直接面對藍新 / 綠界客服 + 處理 callback bug

7. Mass Email + Campaign Tracking

  • 每小時 400 封內建寄信、開信追蹤、campaign tracking
  • 自建要接 SendGrid / Mailgun / Mailchimp + 寫 tracking
用最白話的比喻:netiCRM = 「為台灣 NPO 預先組裝好的 IKEA 家具」 — 比自己劈木頭快、便宜、合規。WordPress + 多 plugin = 「自己劈木頭組」 — 可行但需要木工技能 + 持續維護;省下的錢用在請木工和買工具。
§4 INTEGRATION

AIA 自建站 × netiCRM 的 4 條整合路徑

「自建網站要如何整合 netiCRM?」有 4 條典型路徑,由簡到繁。每條的成本、彈性、Maintainability 不同。

PATH 1 · PRIMARY · 主推

iframe 嵌入(最便宜、最快)

AIA WordPress 站的某頁(例:報名 / 捐款 / 加入會員)直接嵌入 netiCRM 的表單作為 iframe。表單視覺由 netiCRM 主機提供,但 AIA 站的 header / footer / 品牌都保留。

技術需求WordPress 用 Custom HTML widget 貼 iframe code,無需開發
netiCRM 加購0(微型基本方案內建)
視覺一致性中 — iframe 內樣式跟 AIA 主站可能略差,但可接受
中英雙語若要英文版:諮詢 netiCRM 能否複製表單改 label;或買 NT$10,500 雙語加購
URL表單仍在 neticrm.tw/xxx,但 iframe 外殼是 aia.org.tw — donor 看到 AIA 站
Maintainability極高 — Elementor 內貼 embed code 就好
適用第一年 ADALS / 過渡期 — 先用最便宜方式上線,確認 UX 與工作流順暢後再評估升級
PATH 2 · ALTERNATIVE · 替代

自訂域名(donate.aia.org.tw)+ CSS 客製

netiCRM 表單 host 在 AIA 的子域名下,例如 donate.aia.org.tw 或 register.aia.org.tw。donor 不會看到 neticrm.tw,看起來像 AIA 自家頁面。CSS 客製讓表單視覺貼近 AIA 設計。

技術需求DNS CNAME 設定(IT 5 分鐘)+ netiCRM 配合啟用自訂域名
netiCRM 加購自訂域名 首年 NT$6,000 / 年費 NT$2,000
視覺一致性高 — CSS 客製可貼近 AIA 設計
中英雙語建議同時買雙語加購(NT$10,500),否則自訂域名加值有限
URLdonate.aia.org.tw — 完整品牌一致
Maintainability高 — DNS 一次設,後續 netiCRM 維護
適用第二年起 / 品牌一致性需求強 / 海外捐款者多時
PATH 3 · FALLBACK · 進階

REST API + 自建表單(完全客製)

AIA 站自建表單(完全用 Elementor + AIA 設計),透過 netiCRM 的 REST API 把資料寫回 netiCRM 後台。donor 100% 看到 AIA 設計,但表單背後跑 netiCRM。

技術需求需開發工程師 — 寫 API call、處理 callback、串金流流程
netiCRM 加購0(API 內建)— 但開發成本可能 NT$30-50K 外包
視覺一致性滿分 — 完全 AIA 設計
中英雙語由 AIA 站處理(已是雙語的話直接 reuse)
URLaia.org.tw/donate(完全自家)
Maintainability中 — API 改版時要跟著改;工程師離職交接成本
適用第三年以後 / 品牌極度敏感 / 有專職工程師
PATH 4 · BASELINE · 全包

netiCRM 全包(域名 + 雙語 + 收據 + 電子發票 + 客製)

把所有加購買滿,netiCRM 端做最完整的解決方案。AIA 主站只負責「介紹 AIA + 連結到 donate.aia.org.tw」。

技術需求低 — netiCRM 包辦
netiCRM 加購NT$10,500 雙語 + NT$6,000 域名 + NT$3,750-21,250 收據客製 + NT$16,800 電子發票 = NT$37,050-54,550 加購(一次性 + 年費)
視覺一致性高(CSS 客製 + 自訂域名)
中英雙語含
URLdonate.aia.org.tw
Maintainability最高 — netiCRM 一個窗口
適用不缺預算 / 想最少自己折騰 / 有海外大量 donor
推薦演化路徑:第一年 PATH 1(iframe,只買微型基本)→ 第二年若 ADALS 海外比例高再加 PATH 2(域名 + 雙語)→ 第三年若工程資源到位再考慮 PATH 3 局部客製。不建議一開始就 PATH 4 全包 — 投入大但很多功能可能第一年用不到。
§5 QUESTIONS

諮詢問題清單(分類版,給開會用)

本清單分 11 大類,每題標示優先度。優先度說明:

MUST 必問 — 決策依據 SHOULD 該問 — 規劃所需 NICE 加分 — 加深理解

A. 中英雙語 / 在地化 MUST

  1. 「中英雙語建置加購 NT$10,500」具體包含什麼?是一次性 setup 還是年費?
  2. NT$10,500 涵蓋的範圍是「整套對外頁面」還是「只有募款頁 / 報名頁」?後台管理介面包含嗎?
  3. 不買這個加購,能不能複製一份表單、把欄位 label 改為英文?(這樣 AIA 自己分兩份表單給中文 / 英文 donor)
  4. 英中切換的 UX 是「同一頁切換按鈕」還是「兩個獨立網址」?哪一種建議?
  5. 感謝信 / 收據是否能對英文 donor 自動產出英文版?
  6. 後台是否也有英文模式?(對未來找國際志工 admin 有用)
  7. 有沒有客戶實際做雙語的範例網址可以看?

為什麼必問:這項是 NT$10,500 的支出決策依據,且影響 ADALS 海外與會者體驗。

B. 整合方式(iframe / API / WordPress)MUST

  1. iframe 嵌入到 WordPress + Elementor 站的標準流程是什麼?
  2. iframe 是否支援自適應高度(避免內嵌捲軸 + 表單變長時自動撐高)?
  3. 有沒有 JavaScript SDK / Web Component 替代 iframe?(更現代的嵌入法)
  4. 有沒有官方 WordPress plugin?
  5. REST API 文件公開嗎?可否提供連結?
  6. API 有rate limit 嗎?(萬一 ADALS 開賣瞬間湧入)
  7. API 是否支援 webhook 通知(有人捐款 → notify Slack / Email)?
  8. 有自訂子域名(donate.aia.org.tw)的設定方式?DNS 需要 CNAME 還是 A record?
  9. SSL 憑證:自訂域名是 netiCRM 申請 Let's Encrypt 還是 AIA 提供?

為什麼必問:決定 §4 走哪條整合路徑、決定 AIA 工程量。

C. UI 客製範圍 SHOULD

  1. CSS 客製能改到哪些範圍?(顏色、字體、間距、表單欄位順序?)
  2. LOGO 替換是內建還是加購?
  3. 表單欄位能新增 / 刪除 / 重排嗎?有上限嗎?
  4. 確認頁 / 感謝頁 / 失敗頁文字能改嗎?
  5. 自動寄信(感謝信、收據信)的內容、寄件人能改嗎?需要 SMTP 設定?
  6. 付款失敗後能 redirect 回 AIA 站特定頁嗎?
  7. 有沒有 brand kit / theme 工具(上傳 logo + 色碼直接套用)?

為什麼該問:影響 ADALS 報名頁與 AIA 主站視覺一致性。

D. 金流 / Gateway MUST

  1. 預設綁的金流 provider 有哪些?(藍新 / 綠界 / MyPay)
  2. 國際卡支援度?(VISA / Mastercard / AMEX / JCB)費率多少?
  3. 跨幣別?(只能 TWD,還是能開 USD / HKD?)— ADALS 海外 sponsor 需要
  4. LINE Pay / Apple Pay / Google Pay 支援?
  5. ATM 虛擬帳號 / 超商代碼?
  6. 定期定額 recurring(月捐)的 UX?跟一般捐款一個流程嗎?
  7. 信用卡分期?(若 ADALS 高階票價 NT$8,500 想分 3-6 期)
  8. 失敗重試 / dunning 邏輯有嗎?(月捐 churn 處理)
  9. 收款撥款週期?D+? 多久款項入到 AIA 玉山帳戶?
  10. 金流出問題的escalation channel?(藍新 vs netiCRM 誰先處理)

為什麼必問:ADALS 多元支付決定報名轉換率。

E. 收據 / 發票 MUST

  1. 標準收據 template 長什麼樣?能否拿樣本看?
  2. 收據客製 NT$3,750-21,250 的差別在哪?(LOGO、章程、上傳簽名、客製欄位?)範圍上限?
  3. 電子收據與紙本收據兩種都自動寄嗎?還是 AIA 自己印紙本?
  4. 電子發票串接 NT$16,800 涵蓋什麼?是一次性 setup 還是年費?
  5. 我們有需要電子發票(統一發票)嗎?何種情境 NPO 必開?(ADALS 報名費是銷售還是捐款?)
  6. 統一發票(B2B 企業贊助)和捐款收據(個人 NPO)的區分處理邏輯?
  7. 若年度開錯收據可作廢 / 重開嗎?需要客服協助還是自助?
  8. 收據編號自動產生嗎?有沒有編號規則?
  9. 政府(國稅局 / 內政部)有對 NPO 抵稅收據格式新規嗎?netiCRM 跟進的節奏?

為什麼必問:抵稅收據是 NPO 核心,出錯 = 合規風險。

F. NPO 合規(11(4) 認定 + 公益勸募條例)MUST

  1. AIA 還沒拿到所得稅法 §11(4) 認定 + 公益勸募字號,先採用 netiCRM 該怎麼用?哪些功能會是 disabled 狀態?
  2. 取得認定後,啟用抵稅收據的流程是?要重新 setup 還是 toggle 一鍵切換?
  3. 「年度勸募申報報表」具體會產出什麼?能直接交給內政部嗎,還是匯出 Excel 再人工整理?
  4. 公益勸募條例 §10(揭露)/ §14(用途)/ §15(報告)需要的欄位都內建嗎?
  5. 統一編號 handling 是自動驗證格式還是只記錄字串?有沒有對接財政部統編查詢 API?
  6. 個資法 / GDPR 對 EU donor 的處理?(我們未來可能有 EU 國際贊助)
  7. donor 要求刪除個資時的處理流程?(右被遺忘權)

為什麼必問:法規不合規 = donor trust 崩盤 + 主管機關開罰。

G. 資料所有權 / Export / Migration SHOULD

  1. 全部 donor 資料能 export 為 CSV / Excel 嗎?完整度?
  2. 自動備份多久一次?保留多久?
  3. 若未來想搬到其他平台(自建 CiviCRM、Salesforce NPSP),資料 export 完整度?
  4. 跨年資料保留無上限嗎?還是會自動 archive?
  5. 合約終止時資料怎麼處理?有retention 期讓 AIA 拿資料嗎?
  6. API key 管理?能分權限(read-only / write)嗎?
  7. 能 import 既有 donor 名單 Excel 嗎?(若 AIA 已有歷史 contact)

為什麼該問:資料是 NPO 最重要的資產;未來 vendor lock-in 程度。

H. 計費 / 合約 / 試辦 MUST

  1. 微型 NT$12,960/年 + NT$8,000 開通費 + 申請 2 年合約免開通費,正確嗎?
  2. NPO 試辦條件是什麼?有沒有試用期(30 天 / 60 天)?試用結束自動續約還是要手動?
  3. 中途升級方案(微型 → 小型)如何計費?是差價還是重新算?
  4. 中途降級可以嗎?
  5. 年費繳費方式?轉帳 / 信用卡 / 月付?
  6. 若 contact 數超過 3,000(微型上限)會怎樣?自動升級還是停權?有沒有 grace period?
  7. 有沒有「ADALS 一次性活動方案」(只用 3 個月)的特殊定價?
  8. 政府補助 / 公益優惠?

為什麼必問:錢的事不能模糊。

I. Onboarding / Support / SLA SHOULD

  1. 從簽約到上線的標準時程?ADALS 6 月底前需要上線,做得到嗎?
  2. Onboarding 過程中有專人對接嗎?
  3. 開通後客服管道?(電話 / Email / Slack / Discord?)回應時效承諾?
  4. 出問題的 escalation channel?(誰是 ultimate decision-maker)
  5. 系統 SLA?(uptime guarantee 99.x%)
  6. 系統一年維護更新節奏?有預告嗎?會影響服務嗎?
  7. 文件 / 教學影片 / KB 完整度?有沒有公開連結?
  8. 提供 admin 操作訓練嗎?(線上 / 線下 / 一對一)費用?
  9. 有用戶社群 / 論壇嗎?

為什麼該問:選工具不只看功能,還看出事時誰救你。

J. WordPress / Elementor 整合 SHOULD

  1. 有官方 WordPress plugin 嗎?維護中?
  2. WP 站內顯示「最近捐款者牆」、「捐款金額累計進度條」、「會員人數」的方法?需要 plugin 還是手動 API?
  3. 跟 WP 既有 SEO plugin(Yoast / RankMath)有衝突嗎?
  4. 跟 GA4 / Google Tag Manager 整合追蹤?(donor conversion funnel)
  5. 隱私權政策 / Cookie 政策怎麼處理?(GDPR / 個資法 — iframe 跨域 cookie)
  6. 有實際 WP + Elementor + netiCRM 客戶案例 URL 可看嗎?

為什麼該問:AIA 主站建在 WP + Elementor 上,整合摩擦會直接打 Christina 的工。

K. 無障礙(AIA 特殊敏感)MUST

  1. netiCRM 表單頁面是否符合 WCAG 2.2 AA?有沒有測試報告?
  2. 是否支援螢幕閱讀器(NVDA / JAWS / VoiceOver)測試?
  3. 鍵盤可達性?(Tab / Enter / Esc 全流程)
  4. 表單錯誤訊息是否被 screen reader 朗讀?(aria-live region 設定)
  5. 色彩對比度?(對 low vision donor)
  6. 中文 + 英文 lang 屬性處理?
  7. 願意接受 AIA 做a11y 第三方測試並接受改善建議嗎?

為什麼必問:AIA 是 a11y NPO — 自家工具不無障礙會被 community 質疑。

L. 加分題(Nice to know)NICE

  1. Data center 在哪?(資料主權:台灣 / 新加坡 / 美國)
  2. ISO 27001:2022 認證涵蓋範圍?(只是公司還是含 netiCRM 服務?)
  3. 有沒有 penetration test 報告可看?
  4. 跟其他 NPO 平台(MyPay、聯合勸募網)的關係?
  5. 產品 roadmap 公開嗎?未來 12 個月會有什麼新功能?
  6. 有沒有用戶滿意度調查資料可分享?
  7. 客戶數成長狀況(2024-2026 年)?

為什麼加分:這些不會 block 採用,但對長期合作判斷有幫助。

總計約 80 題,實際開會建議:
  1. 會前內部討論 30 分鐘,刪掉 AIA 用不到的(例如:若決定第一年只走 iframe,可跳過 §C 的 LOGO 客製細節)
  2. 剩下約 40-50 題,實際提問會大概 60-90 分鐘
  3. 建議分 2 場開:第 1 場 60 分鐘問 MUST 類 + 整合方式;第 2 場(若 quote 進入細節)再問 SHOULD / NICE
§6 COST-SAVING

採 netiCRM 前提下的省錢策略

本節假設 AIA 已決定採 netiCRM。在這個前提下,以下 8 個招數能讓總成本壓到最低,且不影響核心功能。

招數 1:申請 2 年合約免開通費(省 NT$8,000)

  • 微型方案開通費 NT$8,000,簽 2 年合約直接免
  • 2 年承諾下:NT$12,960 × 2 = NT$25,920(vs 1 年 NT$12,960 + 開通 NT$8,000 = NT$20,960 + 第 2 年 NT$12,960 = NT$33,920)
  • 結論:2 年合約省 NT$8,000(等於一年的開通費)

招數 2:第一年走 iframe(§4 PATH 1),0 加購

  • 不買雙語(NT$10,500)、不買域名(NT$6,000)、不買收據客製、不買電子發票
  • iframe 嵌入到 WordPress,Christina 5 分鐘搞定
  • 第一年總成本:NT$12,960(微型)only

招數 3:中英雙語先用「兩份表單」策略試水(可能省 NT$10,500)

  • 諮詢時問:「能不能複製一份表單、把欄位 label 改為英文,做成兩份表單(中文 / 英文)?」
  • 若可以 → 不買 NT$10,500 雙語加購,AIA 主站做語言 router(用戶選中 / 英 → 連到對應 netiCRM 表單)
  • 若不可以(netiCRM 限制) → 第一屆 ADALS 只開中文表單,英文 donor 用 PayPal 或 invoice 走另一條路
  • 第二年依實際海外 donor 比例再決定是否升級

招數 4:延後買電子發票串接(省 NT$16,800/年)

  • 「ADALS 報名費需開統一發票」這件事先讓會計師 / 國稅局判斷,不是預設要買
  • 若 ADALS 報名費被分類為「非銷售性活動」(NPO 推廣活動),可能不需要
  • 若會計師確認需開 → 再買
  • 不要先入為主買齊

招數 5:用 netiCRM 標準收據 template,不買收據客製(省 NT$3,750-21,250)

  • 標準收據格式合規(內政部要求的欄位都齊),只是沒有 AIA LOGO
  • 第一屆 ADALS donor 收到「合規但無 logo 的收據」可接受
  • 等 AIA 拿到 §11(4) 認定、確認需要客製品牌收據時再加購

招數 6:不買自訂域名,接受 neticrm.tw/xxx 網址(省 NT$6,000 首年 + NT$2,000/年)

  • iframe 嵌入時,donor 在地址列看到的是 aia.org.tw/donate(AIA 主站)
  • 表單內部 action URL 才是 neticrm.tw,donor 不會直接看到
  • 除非品牌極度敏感,可以接受

招數 7:MyPay 作為小額捐款專用第二金流(NT$9/筆 vs 藍新 2.4%)

  • 取得 §11(4) 認定後啟用
  • 單筆 ≥ NT$400 的捐款用 MyPay(NT$9 固定費 < NT$400×2.4% = NT$9.6)
  • 小額用藍新
  • 關鍵:諮詢 netiCRM 時確認 MyPay NT$9/筆條件是否仍適用 AIA(見 payment-stack §7.6 修正版)

招數 8:活動結束後降級(若可行)

  • ADALS 6-9 月衝刺期維持微型方案
  • 10 月起 ADALS 結束、回到日常捐款 / 會員 ops,contact 數成長前可考慮維持微型
  • 跨入第二年若 contact 接近 3,000 上限,再評估升級小型方案
  • 諮詢時問:contact 超過上限時的應對與升級路徑
最省組合(第一年):微型 + 2 年合約 + 0 加購 + iframe 嵌入

第一年總成本 = NT$12,960(年費),0 開通費,0 加購。等於月付 NT$1,080。

vs 全包路徑(§4 PATH 4)第一年約 NT$50K-65K。差距 4-5 倍。第一年用最省組合上線,實際 ADALS 跑完一輪再決定要不要加買哪些加購,風險最低。
