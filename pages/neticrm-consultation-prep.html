QUESTIONS · NETICRM

netiCRM 問題清單

11 大類,本週寄出諮詢用。「中英雙語建置加購 NT$10,500 是什麼」的背景說明見頁尾參考區。

A. 中英雙語 / 在地化 MUST

  1. 「中英雙語建置加購 NT$10,500」具體包含什麼?是一次性還是年費?涵蓋範圍是「整套對外頁面」還是「只有募款 / 報名頁」?

  4. API 支援 webhook 通知嗎?(有人捐款 → notify Slack / Email)

  6. 有 WP + Elementor + netiCRM 的客戶案例 URL 可看嗎?

H-彈性. 暫時 / 活動型 / 月繳方案(AIA 評估重點)

  4. 年費能不能改月繳,分散現金流?
  5. 有沒有「非活動月份降規 / 暫停」的選項?

必問:H-彈性 Q2(停用後資料保留)決定「暫時方案」是真選項還是陷阱 — 若停用後資料被鎖、明年要重建,等於落入「省月費、爆重做工時」。判讀前提:AIA 要先自己確認「有沒有把握年復一年持續辦活動 + 收捐款」。有把握 → 連續性有價值,年費制合理;不確定(ADALS 可能是一次性試辦)→ 用力爭取活動型短期定價。

QUESTIONS · RECUR.TW

Recur.tw 問題清單

這份清單的定位:payment-stack-decision 對 Recur 的論述是「依公開資訊推測」。本清單目的是驗證那些推測 — 若 Recur 回覆顯示已支援 NPO 法規層,主結論需重評。題數刻意比 netiCRM 少,聚焦關鍵未知。建議另外排(非本週承諾)。

R-0. 產品成熟度與穩定度(先問這個)MUST

本頁研究時(2026-05),第三方報導指出 Recur 似乎仍在 alpha 封測、尚未開放公開註冊。一個還在封測的產品,對 ADALS 2026 這種真實活動是 material risk。所以先問清楚:

  1. Recur 目前是否已脫離 alpha / 封測,正式對外開放註冊?產品正式上線多久了?
  2. 標示「coming soon」的功能(會員資料管理、電子發票、評價系統)各自預計何時上線?
  3. 有沒有 SLA(uptime 保證)?出問題的客服 / escalation 管道是什麼?
  4. 目前付費客戶數量級?有沒有跑過大型活動 / 高併發收款規模的實際案例?

必問:若 Recur 仍在封測 → ADALS 2026 直接不考慮它(風險太高)。就算未來想用 Recur 做會員訂閱,也要等它正式 GA + 會員資料管理上線。這題決定後面 R-A 到 R-F 還用不用得上。

R-A. NPO 法規(最關鍵 — 驗證 deal-breaker)MUST

  1. Recur 能否開立台灣 NPO 抵稅收據(捐款人可拿去報稅列舉扣除)?若不能,有沒有 roadmap?
  2. 有沒有電子發票(統一發票)串接?
  3. 企業捐款時,系統能否把統一編號帶到收據 / 發票?
  4. 有沒有公益勸募條例要求的合規欄位(勸募字號、用途、執行狀況)與年度申報資料匯出?
  5. 有沒有台灣 NPO / 公益團體的實際客戶案例?

必問:這 5 題若全是「沒有」,確認 Recur 不適合當主 stack;若有意外的「有」,payment-stack-decision 主結論需重評。

R-B. 費率(驗證本頁推算的 4.9%)MUST

  1. 月營收超過 NT$100,000 之後的費率結構?是否如本頁推算「超出部分加 2.5% 平台費」?
  2. PAYUNi 金流費 2.4% 是否所有方案 / 所有營收區段一致?
  3. 有沒有年費 / 設定費 / 隱藏費用?
  4. NPO / 公益團體有沒有優惠費率?

必問:payment-stack-decision §6 的 break-even 表建立在「超額 4.9%」推算上,需 Recur 官方確認。

R-C. 「WordPress 報名 plugin + Recur」組合可行性(AIA 評估重點)MUST

這是 AIA 想評估的重點。Recur 本身沒有活動報名功能,但若能跟 WordPress 的活動報名 plugin 組合(報名交給 plugin、收款交給 Recur),仍可能服務 ADALS。AIA 與 Recur 創辦人有直接管道,以下問題能拿到權威答案,不必用猜的。

  1. Recur 本身有沒有活動報名功能(報名表單 / 多票種 / 早鳥 / 報到)?若沒有,有沒有 roadmap?
  2. 若沒有 — Recur 能不能當 WordPress 活動報名 plugin 的金流?具體:有沒有 WooCommerce payment gateway plugin?(Event Tickets Plus 等票務 plugin 多半架在 WooCommerce 上)
  3. Recur 能不能處理一次性付款(活動票券),不只訂閱式 recurring?
  4. Recur 的 checkout 能不能嵌進 WordPress 報名流程(Web Component / API),讓「填報名表 → 付款」一條龍,不是跳兩個分開的頁面?
  5. (最關鍵)報名資料在 WordPress plugin、付款資料在 Recur,兩邊怎麼對應 / 對帳?Recur 有沒有 webhook 把付款結果回拋給 WordPress,自動標記「這筆報名已付款」?
  6. 收據由誰開?WordPress plugin、Recur、還是都不開、要 AIA 手動?
  7. 有沒有實際用 WordPress + Recur 跑活動售票的案例(台灣 NPO 尤佳)?

必問:Q5(對帳)是這個組合的成敗關鍵。若 Q2-Q5 答案乾淨(有 WooCommerce gateway、能嵌入、webhook 自動對帳)→ WordPress plugin + Recur 是 netiCRM 之外的真實替代;若 Q5 要人工對帳 → 這條路落回「跨系統對帳的拼裝」,違反維護性原則。

R-D. 其他整合 SHOULD

  1. Cal.com / Circle 是否有原生整合?(payment-stack-decision §2 標示「未經官方驗證」,需確認)
  2. Webhooks / Zapier / Make 之外,還有哪些官方整合?
  3. API 文件公開嗎?有 rate limit 嗎?

該問:確認 payment-stack-decision §2 spec 表對 Recur 整合能力的描述是否正確。WordPress 嵌入相關問題已併入上方 R-C。

R-E. 國際付款 / 雙語 SHOULD

  1. 支援海外信用卡 / 跨幣別(USD / HKD)嗎?
  2. 有沒有英文 / 中英雙語的付款頁面?
  3. 海外付款者的收據 / invoice 處理?

該問:若 AIA 評估「creator economy 副業」用 Recur,海外收款能力是關鍵。

R-F. 資料 / 合約 NICE

  1. donor / 訂閱者資料能 export 為 CSV / Excel 嗎?
  2. 中途終止合約,資料如何處理?有 retention 期讓 AIA 拿資料嗎?
  3. 月營收上限 / contact 數上限?

加分:評估 vendor lock-in 與未來退出成本。

Recur 諮詢的判讀原則:
  • R-A(NPO 法規)若大致都是「沒有」→ 確認 Recur 不能單獨當 NPO 主 stack。若出現意外的「有」→ payment-stack-decision 主結論需重評。
  • R-C(WordPress plugin + Recur 組合)是另一條評估線:就算 Recur 自己沒 event,只要 Q2-Q5 答案乾淨(有 WooCommerce gateway + 能嵌入 + webhook 自動對帳),這個組合就是 netiCRM 之外的真實選項;若 Q5 要人工對帳,則回到「不推薦的拼裝」。
  • R-B(費率)用來驗證 §6 break-even 表的 4.9% 推算。

背景參考(預設收合)

以下是支撐問題清單的背景說明。開會前不一定要讀,開會後對照用。

「中英雙語建置加購 NT$10,500」是什麼 EXPLAINER

netiCRM 欄:NT$10,500 = 一次性「建一個英文版」服務費

  • 預設狀態:netiCRM 微型方案買來後,所有對外頁面(募款頁、報名頁、收據)都是繁中 only,後台管理介面也是繁中。
  • NT$10,500 涵蓋什麼(依公開資訊推測,須諮詢確認):netiCRM 替 AIA 建一份「英文版對外頁面」,讓海外捐款者 / 報名者用英文流程。技術上是 CiviCRM 多語模組啟用 + 內容翻譯 + UI 英文化。
  • 不涵蓋:後台管理介面(仍繁中,給工作人員用)。
  • 一次性還是年費?不確定 — 要問(問題 A-1)。

Recur.tw 欄:「未提」= 公開資料沒寫

「未提」不是「沒有」也不是「有」,是 Recur 公開定價 / spec 頁沒明確提到雙語。本週諮詢 Recur 時順便問(問題 R-E-2)。

AIA 該不該買這 NT$10,500?三個判斷關鍵:① ADALS 海外與會者比例(<10% 可能不需要,>30% 明顯加分);② 「英文版」涵蓋範圍(只要表單 vs 整頁含收據);③ 有沒有「複製表單改 label」的免費替代(問題 A-2)。
AIA 自建網站怎麼整合 + netiCRM 的價值 整合路徑 / 7 項價值

AIA 現況

主站 = WordPress + Elementor;有既定 Style Guide;目標 WCAG 2.2 AA + 中英雙語。netiCRM 不是來替代主站,是來當主站的金流 / 報名 / CRM 後端 — AIA 站負責品牌敘事,netiCRM 負責「按鈕一按開始繳費 / 開收據 / 進 CRM」。

4 條整合路徑(由便宜到完整)

PATH 1 · 主推

iframe 嵌入(最便宜、最快)

把 netiCRM 表單以 iframe 嵌進 WordPress 頁。0 加購、Christina 5 分鐘搞定、AIA 品牌外殼保留。視覺一致性中等,donor 地址列看到 aia.org.tw。第一年 / 過渡期主推。

PATH 2 · 替代

自訂域名(donate.aia.org.tw)+ CSS 客製

netiCRM 表單 host 在 AIA 子域名,donor 不會看到 neticrm.tw。需自訂域名加購(首年 NT$6,000 / 年費 NT$2,000)。視覺一致性高。第二年起 / 品牌一致性需求強時。

PATH 3 · 進階

REST API + 自建表單(完全客製)

AIA 站自建表單,透過 API 寫回 netiCRM。視覺 100% AIA 設計,但需開發工程師(估外包 NT$30-50K)。第三年以後 / 有專職工程師時。

PATH 4 · 全包

netiCRM 全包(域名 + 雙語 + 收據 + 電子發票)

所有加購買滿,netiCRM 包辦。加購合計 NT$37,050-54,550。不缺預算 / 想最少自己折騰時。

netiCRM 對 AIA 的 7 項價值(vs 全自建)

  1. 台灣 NPO 抵稅收據 — 合規格式內建,取得 §11(4) 認定後一個 toggle 就 unlock。自建要找會計師 + 工程師對接,估 1-3 個月。
  2. 公益勸募條例合規欄位 + 年度報表 — 內建。自建要研讀條例 + 改表單 + 寫匯出。
  3. 統一編號 handling — 企業捐款者統編自動帶到收據。
  4. 跨年 donor / 會員 CRM — 捐款歷程、互動紀錄、會員資格。自建只能用 Google Sheets,50 人後變負擔。
  5. Event Registration — 報名頁、QR 簽到、優惠碼、多票種、報名+繳費一條龍。
  6. 已串好的金流 — 不用自己接金流 SDK,出問題 netiCRM 客服可協助。
  7. Mass Email + Campaign Tracking — 每小時 400 封內建寄信、開信追蹤。
白話比喻:netiCRM = 「為台灣 NPO 預先組裝好的 IKEA 家具」,比自己劈木頭快、便宜、合規。WordPress + 多 plugin = 「自己劈木頭組」,可行但需要木工技能 + 持續維護。
省錢策略 + 建議的第一年 MVP 配置 採 netiCRM 前提下

省錢策略(採 netiCRM 前提下,8 招)

  1. 申請 2 年合約免開通費 — 省 NT$8,000。
  2. 第一年走 iframe(PATH 1),0 加購 — 第一年總成本只有 NT$12,960。
  3. 雙語先用「兩份表單」試水 — 諮詢能否複製表單改 label,可能省 NT$10,500。
  4. 延後買電子發票串接 — 先讓會計師 / 國稅局判斷是否必開,省 NT$16,800/年。
  5. 用標準收據 template — 不買收據客製,省 NT$3,750-21,250。
  6. 不買自訂域名 — 接受 neticrm.tw 子網址,省 NT$6,000 首年 + NT$2,000/年。
  7. MyPay 作小額捐款第二金流 — 取得認定後,單筆 ≥ NT$400 用 MyPay 較省(條件見 payment-stack §7.6)。
  8. contact 數接近上限再升級 — 維持微型方案到接近 3,000 contact 才評估升級。
