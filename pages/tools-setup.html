無障礙測試工具安裝教學

返回專案文件 前往測試流程

開始測試前，請先安裝以下工具

全部免費。安裝完成後即可使用 WCAG 2.2 測試流程進行測試。預估安裝時間：15-20 分鐘。

1. axe DevTools

Chrome 擴充 ▶

是什麼

業界最廣泛使用的自動化無障礙掃描工具。能檢測出約 30-40% 的 WCAG 問題，是測試的第一步。

安裝

Chrome Web Store → 安裝 axe DevTools

使用方法

  1. 打開要測試的網頁
  2. 按 F12 開啟 DevTools
  3. 在上方分頁列找到 axe DevTools（可能需要按 >> 展開更多分頁）
  4. 點擊 Scan All of My Page
  5. 等待掃描完成，查看結果列表
  6. 每個問題會標示嚴重程度（Critical / Serious / Moderate / Minor）
免費版已經非常夠用。不需要登入或付費。掃描結果可以直接截圖貼到測試流程頁面。

2. Lighthouse

Chrome 內建 ▶

是什麼

Google 內建的網頁品質檢測工具，會給無障礙一個 0-100 的分數。跟 axe 互補——Lighthouse 給整體分數，axe 給細節。

安裝

不需要安裝，Chrome 已經內建。

使用方法

  1. 打開要測試的網頁
  2. 按 F12 開啟 DevTools
  3. 找到 Lighthouse 分頁
  4. Categories 只勾選 Accessibility（其他取消勾選）
  5. 點擊 Analyze page load
  6. 等待完成，記錄 Accessibility 分數
目標分數：90 分以上。如果低於 90，先看 axe 的詳細報告找出問題。

3. WAVE

Chrome 擴充 ▶

是什麼

WebAIM 開發的視覺化無障礙檢測工具。最大特色是直接在網頁上標記問題位置，用顏色和圖示告訴你「這裡有問題」。比 axe 更直觀。

安裝

Chrome Web Store → 安裝 WAVE

使用方法

  1. 打開要測試的網頁
  2. 點擊瀏覽器工具列的 WAVE 圖示
  3. 頁面會出現各種標記：
    • 紅色 = Error（必須修）
    • 黃色 = Alert（需要人工判斷）
    • 綠色 = Feature（做得好的地方）
    • 紫色 = Structural（結構資訊）
  4. 點擊任何標記可以看詳細說明
WAVE 的「Contrast」分頁特別好用——會列出所有對比度不足的元素，不需要一個一個用 CCA 手動測。

4. Colour Contrast Analyser (CCA)

桌面應用程式 ▶

是什麼

TPGi 開發的對比度檢測工具。用滴管取色，即時顯示對比度數值和 Pass/Fail 結果。測對比度最精確的工具。

安裝

TPGi 官網 → 下載 CCA

選擇 Windows 或 Mac 版本，下載安裝。

使用方法

  1. 開啟 CCA
  2. Foreground（前景色）：點滴管圖示，在網頁上點擊文字
  3. Background（背景色）：點滴管圖示，在網頁上點擊文字後面的背景
  4. 查看結果：
    • 一般文字：需要 ≥ 4.5:1
    • 大文字（18pt 以上 / 14pt 粗體以上）：需要 ≥ 3:1
    • 非文字 UI 元件（按鈕邊框、icon）：需要 ≥ 3:1
測對比度的時候，別忘了測 hover 和 focus 狀態的顏色——滑鼠移上去或 Tab 選到時顏色可能會變。

5. HeadingsMap

Chrome 擴充 ▶

是什麼

顯示網頁的標題結構樹（H1-H6）。一眼看出標題有沒有跳級、有沒有多個 H1。

安裝

Chrome Web Store → 安裝 HeadingsMap

使用方法

  1. 打開要測試的網頁
  2. 點擊瀏覽器工具列的 HeadingsMap 圖示
  3. 會顯示整頁的標題樹狀結構
  4. 檢查：
    • 只有一個 H1
    • 層級遞進：H1 → H2 → H3（不跳級）
    • 標題文字有意義（不是「Section 1」「Section 2」）
Elementor 中最常見的問題：為了讓文字看起來大/小而選錯標題層級。字體大小應該用 Style 設定，不是改 H tag。

6. Landmarks

Chrome 擴充 ▶

是什麼

顯示網頁的地標區域（header, nav, main, footer 等）。螢幕閱讀器使用者用地標快速跳轉到頁面不同區域。

安裝

Chrome Web Store → 安裝 Landmarks

使用方法

  1. 打開要測試的網頁
  2. 點擊瀏覽器工具列的 Landmarks 圖示
  3. 會在頁面上框出地標區域
  4. 確認至少有：main（主內容區）
  5. 理想狀態：header + nav + main + footer

7. VoiceOver（Mac）

Mac 內建 ▶

是什麼

Mac 內建的螢幕閱讀器。視障者用它來「聽」網頁內容。測試時開著它瀏覽網頁，就能知道視障者的體驗如何。

安裝

不需要安裝，Mac 已經內建。

開啟 / 關閉

按 Cmd + F5 開啟，再按一次關閉。

基本操作

  1. VO 鍵 = Ctrl + Option（以下簡寫 VO）
  2. VO + 右方向鍵：移到下一個元素
  3. VO + 左方向鍵：移到上一個元素
  4. Tab：跳到下一個互動元素（連結/按鈕）
  5. VO + U：開啟 Rotor（可選擇用 Headings / Links / Landmarks 瀏覽）
  6. H：跳到下一個標題
  7. Enter 或 VO + Space：點擊/啟動元素

測試重點

  • 頁面載入時聽到的標題是否正確？
  • 用 H 鍵跳標題，順序合理嗎？
  • 圖片有聽到 alt text 嗎？裝飾性圖片被跳過了嗎？
  • 連結文字聽起來有意義嗎？（不是一堆「了解更多」）
  • 整體感受：閉著眼睛你知道自己在哪裡嗎？
不需要精通 VoiceOver。開著聽 5-10 分鐘就能發現大部分問題。重點是「聽起來有沒有意義」。

8. NVDA（Windows）

免費下載 ▶

是什麼

Windows 上最廣泛使用的免費螢幕閱讀器。功能完整，全球超過 20 萬人使用。

安裝

NV Access 官網 → 下載 NVDA

下載後執行安裝程式，選擇「Install on this computer」。

開啟 / 關閉

開啟：Ctrl + Alt + N。關閉：Insert + Q，然後按 Enter 確認。

基本操作

  1. NVDA 鍵（預設 = Insert，筆電可改為 Caps Lock）
  2. 下方向鍵：讀下一行
  3. 上方向鍵：讀上一行
  4. Tab：跳到下一個互動元素
  5. H：跳到下一個標題
  6. D：跳到下一個地標
  7. Insert + F7：開啟元素清單（標題/連結/地標）
  8. Enter：點擊/啟動元素

測試重點

跟 VoiceOver 相同——聽標題、聽圖片、聽連結、整體感受。

第一次用會不習慣。先在本站（專案總覽頁面）練習 5 分鐘，熟悉基本操作後再去測 AIA 網站。

安裝確認清單

全部打勾就可以開始測試了。

  • axe DevTools -- Chrome 擴充已安裝
  • Lighthouse -- Chrome 內建，確認 DevTools 能找到分頁
  • WAVE -- Chrome 擴充已安裝
  • Colour Contrast Analyser (CCA) -- 桌面應用已安裝
  • HeadingsMap -- Chrome 擴充已安裝
  • Landmarks -- Chrome 擴充已安裝
  • VoiceOver (Mac) 或 NVDA (Windows) -- 能正常開啟和關閉
