報名與付款整合說明

本頁為前端範本,實際報名邏輯由 netiCRM 提供。

整合說明

採用模式 (b):iframe / embed

點擊「立即報名」後,使用者跳轉至該活動的「活動報名頁面」 (範本見 event-template-form-payment.html), 該頁面在報名區塊嵌入 netiCRM 表單 iframe。優點:

  • No code — netiCRM 後台建立活動後產生 embed URL,WordPress 端只需貼 iframe
  • 付款流程由 netiCRM 處理 — 預計支援藍新金流(NewebPay)+ PayPal
  • 會員資料、報名記錄、收據統一進 netiCRM 後台,無需另建系統
  • 視覺差異可用自訂 CSS 在 netiCRM 後台 override,讓表單樣式接近本站

付款選項顯示: 藍新金流 PayPal

a11y 規範

嵌入 netiCRM 表單時必須確認

  • iframe 必須加 title 屬性(例:「ADALS 2026 報名表單」)
  • iframe 內表單欄位都有 <label>,且必填用 aria-required="true"
  • 錯誤訊息使用 role="alert" 即時通知讀屏使用者
  • 付款選項(藍新 vs PayPal)用 <fieldset> + <legend> 包裹
  • 切換語言時 iframe 來源 URL 也要切換(中文 / 英文版表單)
