§1 PURPOSE

為什麼這頁存在

APAC AIA 從 ADALS 2026 開始將會持續舉辦活動、收會員費、收捐款。這些都涉及收款,而收款的選擇牽涉:

  • 合規:台灣稅法收據、社團法人格式、捐款扣抵編號
  • 跨境:海外贊助、海外個人購票、外幣 invoice
  • 長期 ops:會員資料、跨年度追蹤、自動寄信
  • 預算:NT$20K/年(netiCRM)vs WordPress 套件授權成本 vs 各方案的金流費率

這些屬於組織治理層級的決定,涉及預算編列、人力配置、合規責任、長期分工原則。本頁完整列出考量過的選項、刪除理由、保留理由,讓祕書長掌握運營層細節、常務理事能提出想法與評估可行性、理事長有充分依據做策略決定。

姊妹頁(不在本頁討論範圍): 報名頁的 UI 規格、表單欄位、確認頁、狀態示範等視覺設計,屬於建置層工作,記錄於另一份文件 → 報名流程 UI 設計範本。本頁不重複論述。
§2 TL;DR

結論先 — 三條路徑 Price + Effort + Risk 比較

推薦 stack:玉山銀行(底層,可選兆豐第二帳戶)+ netiCRM(上層 CRM)+ SWIFT(海外大額贊助)。詳細比較如下,刷卡費率四條都一樣(藍新 2.4%),差別在固定成本、setup 投入、長期風險、人工 burden。

價格與時程皆為粗估範圍。

PATH A · PRIMARY · 主推

netiCRM 一站式 NPO 平台

Price · 差異化成本

  • 第一年:NT$20,960(NT$12,960 年費 + NT$8,000 開通費)
  • 第二年起:NT$12,960/年(2 年合約可免開通費)
  • 一次性工程成本:0
  • 條件性額外費用:若會計師判斷 ADALS 票務需開立電子發票 → + NT$25,800 第一年(NT$16,800 netiCRM e-invoice 安裝費 + NT$3,000 ezPay 開通 + NT$6,000/年);上方 NT$20,960 不含此費用

Effort

  • 諮詢:1 週內首次溝通
  • Setup:5-7 天(後台 + 嵌入 AIA 站)
  • Ongoing:每場活動 2-4 小時建置
  • 學習曲線:中(1-2 週熟悉後台)

Risks

  • 諮詢結果可能 NT$20K 方案不適合 → 退 Path B
  • 雙語 / wrap 客製範圍諮詢前未驗證
  • NPO 條件談判實際費率可能跟 NT$20,960 有差
  • 跨年資料 export / migration 難度未驗證
  • Vendor 風險:對 netiCRM 一家公司單點依賴(vendor 停服 = 整 stack 失效,SPOF 集中)

隱性成本

  • 資料搬家費:若 netiCRM 停服或 AIA 未來想換平台,資料搬出去的費用和時間沒有預先估過
  • 學習新系統:第一個月執行小組要花時間熟悉後台操作,估 1-2 週
PATH B · ALTERNATIVE · 替代

WordPress + 套件半 no-code

Price · 差異化成本

  • Plugin 授權第一年:NT$11,000-23,000(依是否需 e-invoice / 會員套件)
    • Elementor Pro Essential(elementor.com) · NT$1,900/年 · USD $59 · 1 site · Confidence 高(已驗證)
    • Events Calendar Pro + Event Tickets Plus(theeventscalendar.com) · NT$9,250/年 · USD $289 · Confidence 高(已驗證 — 2026/02 起 $289 bundle;ADALS 多票種需兩個 plugin 合購)
    • Memberships 套件(若需要) · NT$0-12,000/年 · Confidence 低,看選哪個
      • Paid Memberships Pro:有免費版
      • Restrict Content Pro:~NT$3,000-7,500
      • WooCommerce Memberships:~NT$6,000
      • MemberPress:~NT$5,400-12,000
    • 藍新 ezPay 電子發票(若會計師判斷需要) · Confidence 高(已驗證)
      • 一次性開通費:NT$3,000(直接向 ezPay 申請)
      • 年費制 Basic:NT$6,000/年(500 張/月以內)
      • 是否需要開立電子發票(統一發票),依 ADALS 票務課稅性質由會計師判斷
  • 一次性工程協助 setup:估 NT$30-50K
  • 第二年起:NT$11,000-23,000/年

Effort

  • Plugin 評估:1-2 週逐個 vet
  • Setup + 串接:1-2 週(需工程協助)
  • 收據 e-invoice + 會員:額外 2 週
  • Ongoing:多 plugin 維護,每月 2-4 小時
  • 學習曲線:中(WooCommerce + plugin stack 整合需新學)

Risks

  • Vendor 風險:對多 plugin 廠商各自依賴(關鍵 plugin 停更 = 對應功能層失效,SPOF 分散但仍存在)
  • NPO 收據合規高 risk(自製不一定過財政部規格)
  • 多 plugin 同時更新可能 break
  • 跨年資料分散 4-5 個 DB,migration 困難
  • 沒 NPO 客服 channel,出事自己解
  • 授權年費若不續 = 功能下架

隱性成本

  • 每月維護工時:每個月要花 2-4 小時更新插件、測試沒有壞掉 — 要有人指定負責
  • 活動期間壞掉的緊急修理費:ADALS 報名期(6-9 月)若插件衝突導致報名頁失效,緊急找工程師修比平時貴 2-3 倍
  • 每年重設活動流程:沒有「複製上一年」功能,第二屆 ADALS 要從頭設定一遍,估 3-5 天工時
  • 未來換平台的搬家費:資料散在 4-5 個不同地方,整合搬家費用不低
  • 信用卡資安責任:WordPress 環境的信用卡資安規範要自己確保,netiCRM + 藍新 幫你做掉這層
PATH D · BASELINE · 最低成本

純免費 stack(0 預付 + 大量人工)

Price · 差異化成本

  • Plugin 授權:0 元(全用 free 版)
  • 一次性 setup:0 元(僅內部時間)
  • 第二年起:0 元
  • per-transaction 費(藍新 / PayPal / SWIFT)仍要付,同其他三條

Effort

  • Setup:估 3-5 天(WordPress + free plugins 拼裝)
  • Ongoing:每筆訂單祕書長 / 總務手工開收據 + Excel 管理會員
  • ADALS 規模(200 張)估手工開單 1-2 週工時
  • 學習曲線:中(但 free 版功能限制較多,設計彈性低)

Risks

  • Events Calendar free 版無多票種 / 早鳥規則 → ADALS 多票種需求需 hack 或無法實作
  • 收據格式全靠人工把關 NPO 合規 → 出錯 risk 高
  • 跨年 ops 不可持續(Excel 第二屆要重建)
  • Elementor free 版設計彈性受限 → 視覺品質受影響
  • Vendor 風險:同 Path B(各 plugin 廠商依賴),且 free 版可能停更或 EOL 速度更快

適合:小型活動(< 30 人 workshop / community 聚會)、0 預算嚴格限制的場合 · 不適合 ADALS 規模(200 人 + 多票種,人工 burden 與功能受限都過重)

隱性成本

  • 收據人工工時:200 張收據每張約 15-20 分鐘,合計估 50-70 小時祕書長/總務人力
  • 每月對帳工時:每月 2-4 小時手工核對銀行對帳單
  • 手誤的財稅風險:手工處理出錯率高,收據格式錯誤可能引發財稅問題
PATH C · 寫程式自建(已排除,點擊看細節)

Price · 一次性 + ongoing

  • 一次性 first build:估 NT$300-500K(senior 工程師 6-8 週外包)
  • Server hosting:估 NT$2,000-5,000/年
  • Ongoing 工程維護:估 NT$50-100K/年

為什麼不推薦

  • 工程資源依賴(人離開 = 系統無人維護)
  • 合規 risk 最高(自寫 NPO 收據易違規)
  • Callback bug = 重複扣款 / 漏扣
  • Security 自負(PCI / SSL / XSS)
  • 第二屆要再建,無 reusable infrastructure
  • 沒 vendor escalation channel

隱性成本

  • 工程師離職的交接費:程式沒有文件,新工程師要從頭看懂
  • 每加新功能都要請工程師:第二屆 ADALS 加早鳥規則、退款等都要再付開發費
  • 緊急修理費:出事時緊急叫人比平時貴 2-3 倍
共同的 per-transaction 費用(三條一樣):
  • 國內刷卡:藍新 2.4%(可申請)/ 2.75%(標準)
  • 國際刷卡:PayPal 4.4% + USD$0.30/筆
  • 跨境贊助:SWIFT 手續費 USD 40-80/筆
  • 玉山銀行帳戶:免費,無年費
排序(由低到高):
  • Price(預付 + 年費):D(0)< A < B << C(高一個量級)
  • Effort + Risk(對 ADALS 規模 200 人):A < B << D < C
    • D 在 ADALS 規模下 effort/risk 反而高 — 多票種卡關 + 200 張人工開單 burden
  • Effort + Risk(對小型活動 < 30 人):D < A < B << C
    • D 對小型活動是 cleanest baseline

→ Path D 是條件性選項:不適用 ADALS 規模,但適合 APAC AIA 未來辦小型 workshop / community 聚會時的 0 成本 baseline。

§3 SCENARIOS

4 種收款情境

不能用同一個工具解決所有情境。每個情境的金額規模、付款者類型、合規要求都不同:

台灣付費門票
NT$1,500–8,500 · 個人 · 集中
藍新金流(信用卡 + ATM + 超商)
國外付費門票
USD 60–500 · 個人 · 少量
PayPal Business 或 netiCRM 國際卡
台灣企業贊助
NT$50K–300K · 公司 · 一次性
玉山銀行匯款 + Invoice(不上線)
國外企業贊助
USD 1,500–10,000 · 公司 · 一次性
SWIFT 電匯 + 英文 Invoice(不上線)
關鍵設計原則:贊助走 invoice + 匯款,門票走線上刷卡。**網站只負責「門票」流程**,贊助是「洽詢表單 → 祕書長 / 總務開立 invoice → 線下匯款」三步走。
§4 BANK

銀行帳戶 — 第一順位

現況問題

APAC AIA 目前只有郵局帳戶。郵局帳戶不足以支撐:

不足點影響
不接受國外電匯入帳國外贊助無法收款
大多數金流商不支援郵局領款綠界、藍新、PayPal 都需綁定一般銀行
大額企業匯款處理較慢台灣企業贊助體驗差
國際 SWIFT 訊息流不順跨境電匯易卡關

推薦銀行配置

第一帳戶

玉山銀行 ✓

  • **唯一可綁 PayPal 領款的台灣銀行**
  • 英文網銀介面友善(對國外贊助商查 SWIFT 資訊有用)
  • 對 NPO 開戶經驗較多
  • 台北車站附近有分行(凱薩飯店一帶)

開戶人:理事長 Claire 親辦(法定代表人簽署)。預估時程 2-3 週。

第二帳戶(可選)

兆豐銀行

  • 公股銀行,SWIFT 處理快、手續費較低
  • 大額國際電匯首選
  • 但無法綁 PayPal

策略:若預算與時間允許,讓國外贊助商選 SWIFT 時優先匯到兆豐(費用較低);其他狀況走玉山。

其他考慮過但未選的銀行

中國信託 / 國泰世華 / 富邦 △ — 一般協會常用,服務齊全,但無玉山的 PayPal 綁定優勢,國際電匯費用普遍。沒有不選的理由,但也沒有特別選的理由。若 Claire 已有與某銀行的關係可作第一帳戶,玉山可降為第二帳戶補 PayPal 綁定缺口。

§5 WHY NETICRM

為什麼 netiCRM 是主推

三層結構性整合(Eventbrite 處理不了的)

  1. 台灣稅務無縫對接 — netiCRM 內建符合台灣稅法的收據格式(統編、捐款扣抵編號、社團法人格式),NPO 法人會計直接可用。Eventbrite 給的是美式收據,要另外手動補開台灣格式收據 — 雙重工時 + 合規風險。
  2. 海外付款者整合 — netiCRM 內建國際卡(底層藍新)讓海外與會者用本地信用卡付款;若需要 PayPal UX,可另加 PayPal Business 作為 alternative checkout。後台統一進 NPO 帳戶並產生對應收據。Eventbrite 的 Taiwan organizer payout 不在 Pro 支援名單。
  3. 會員資料跨年度持續使用 — 會員年費、贊助履約追蹤、捐款歷程、自動寄信、會員資格自動審查 — 這些 NPO 標準工作流程 netiCRM 內建,Eventbrite 沒有(它是售票平台,活動結束資料就斷了)。

費率比較(Eventbrite vs netiCRM + 藍新)

票價情境EventbritenetiCRM + 藍新差距
學生票(~NT$2,000)9.3%7.6%netiCRM 低 1.7 pp
一般票(~NT$4,500)7.8%4.7%netiCRM 低 3.1 pp
高階票(~NT$8,500)7.2%3.6%netiCRM 低 3.6 pp

註:% 假設 ~200 張銷售規模。netiCRM 固定年費 NT$20,960 對銷量 dilution 後反映在 %。

但純看 % 差距其實低估了 netiCRM 的真實優勢。真正的價值在於:每年自動產出合規收據、跨國刷卡無縫、會員資料屬於 APAC AIA 自己。即使在學生情境(% 差距最小)下,netiCRM 仍勝出。

netiCRM 真實費率與其他細節已彙整於 §2 TL;DR Path A · Price 區塊。官方 pricing。

§6 USE CASES

4 個 use case lookup 比較

每個 use case 的 top 2-3 方案完整比較,可作為查找用:

Use case 1:國外大額匯款(企業贊助 USD 1,500-10,000+)

方案優點缺點
SWIFT 電匯到玉山(主推)直接收款、合規簡單、無中介雙方手續費共 USD 40-80;贊助商需操作 SWIFT
SWIFT 電匯到兆豐(備用)公股銀行 SWIFT 處理快、手續費較低不能綁 PayPal,需另開戶
PayPal Business(僅小額)贊助商熟悉、操作簡單4.4% + USD 0.30,大額不划算(USD 5,000 吃 USD 220+)

→ 第一屆 SWIFT 玉山為主;有大額或量增評估開兆豐第二帳戶。

Use case 2:國外購票(海外個人 USD 60-500)

方案優點缺點
netiCRM 內建國際卡(主推,若已採 netiCRM)一站式、收據自動、跟報名 / 會員同流;不需另外申請 PayPal BusinessUI 預設繁中,英文 / 雙語切換 + AIA 官網 wrap 嵌入能力諮詢時須確認(底層 CiviCRM 支援多語,但 netiCRM hosted 暴露範圍未驗證);國際卡費率高於國內 2.4%(諮詢時確認)
PayPal Business(備案,綁玉山領款)海外 UX 熟悉、信任度高、台灣協會可申請;不採 netiCRM 時是主方案4.4% + USD 0.30(高於 netiCRM 國際卡)、跟 netiCRM 是兩個 stack 要對帳
Stripe via 美國 LLC(第二屆評估)費率 2.9%、開發友善、UX 國際標準美國 LLC 設立 USD 200-500 + 年維護 USD 100-300、跨境會計複雜

→ 第一屆採 netiCRM:用 netiCRM 內建國際卡為主,PayPal Business **不必申請**;若海外與會者預期多 + UX 反饋差,再加 PayPal 為 alternative checkout。不採 netiCRM(走備案):PayPal Business 為主方案。

Use case 3:國內購票(台灣個人 NT$2,000-8,500)

方案優點缺點
netiCRM(內建藍新,主推)報名 + 收款 + 收據 + 會員一站、台灣付款方式齊全(信用卡 / ATM / 超商代碼)NT$20,960/年(第一年)+ 透過底層藍新 2.4%
WordPress + 套件(替代,Path B)整合在 AIA 既有 WordPress 站、可沿用既有 Elementor 設計流程、藍新 2.4%多 plugin 維護、NPO 收據需另接、Member workflow 不及 netiCRM 直觀
綠界 ECPay市佔最高、SDK 成熟信用卡分期需 NT$10,000/年特約會員、費率 2.88%、市場負評多
LINE Pay 商家版(補強選項,非獨立 stack)台灣用戶熟悉度極高、申請相對快、對 LINE OA 通知整合順純 gateway 不含報名 / 收據 / 會員 — 須搭主 platform(netiCRM / WordPress 等);NPO 商家申請流程須驗證;不支援國際付款者

→ 第一屆 netiCRM 主推;若諮詢結果不採用 → Path B(WordPress + 套件);不建議綠界。Path C(寫程式自建)不在考慮範圍。

Use case 4:國內捐款(個人 / 會員,未來 use case)

方案優點缺點
netiCRM(主推)自動產合規收據(統編 + 捐款扣抵編號 + 社團法人格式)、會員整合、自動寄收據系統年費(若 ADALS 已採用,邊際成本 0)
MyPay(備選)固定手續費 NT$9/筆(對小額極省)、專為 NPO 捐款設計、netiCRM 已整合主捐款流程,不適合報名 / 購票場景

→ 第一屆 netiCRM;未來若想優化小額捐款,可在 netiCRM 內加 MyPay 為第二金流。

已刪除的選項與原因
  • Stripe Taiwan(直接開戶)✗ — Stripe 至 2026/04 仍不接受台灣本地實體開戶(公司、行號、協會、個人皆不可)。台灣不在 Stripe 支援的 46 個地區內。
  • Wise Business ✗ — Wise 官方文件明確列示,慈善與信託組織的商業帳戶服務只支援 EEA、加拿大、美國、瑞士、澳洲、紐西蘭。台灣協會不在支援名單。
  • Payoneer ✗ — 主要針對「個人收款」(自由工作者、跨境電商賣家),企業 / 協會用途的支援與 Wise 類似有地區限制。對 NPO 法人的支援不明朗,且台灣 NPO 使用案例極少。
  • 透過境外實體(美國 LLC / 香港公司)中轉 △ — 註冊美國 LLC 後可開 Mercury 銀行 + Stripe US。但對單場會議不划算(設立 USD 200-500 + 維護 USD 100-300/年 + 跨境會計複雜)。第二屆規模化再評估。
  • 銀行直接金流(中信、玉山等)✗ — 銀行對非營利協會的金流申請門檻嚴(中信要求資本額 500 萬以上、近四個月平均營業額 30 萬以上、實體商品)。協會型 NPO 不符合條件。
  • TapPay ✗ — 純信用卡 token 化解決方案,沒有 ATM、超商代碼、虛擬帳號等台灣本地常用付款方式。功能子集太窄。
§7 PAYMENT DIFF

跨國 / 線下 / 線上 / Hybrid 付費差異

同樣的活動規模,不同的舉辦形式 + 觀眾來源,會直接影響付款基礎建設需求:

維度 線下 (Offline) 線上 (Online) Hybrid 跨國 (International)
主要金流 線上預付 + 現場補單(現金/轉帳) 100% 線上(無現場退路) 兩條並行,需各自 SKU 必須 PayPal / 國際卡(NewebPay 不支援國際付款者)
票種結構 1–2 種(早鳥 / 一般) 通常 1 種 至少 2 SKU(現場 / 線上) 多幣別 SKU(USD / TWD)
收據格式 可選擇實體 / 數位 數位 only 兩種選項並存 必須英文(統編改 Tax ID)
容量管理 場地硬上限 幾乎無限(看頻寬) 兩條容量分管 取決於線上 / 線下混合比例
退費處理 涉及實體成本(已定餐、已印名牌) 簡單(數位資產可回收) 兩條政策可能不同 跨境匯費吃成本(USD 50+ 不退費)
不需要 國際卡(若只給台灣) 場地金流硬體 純現場票流程 純台灣 NewebPay(對國際付款者無感)
特別風險 現場報到混亂、現金管理 連結漏寄、Zoom session limit 兩邊體驗不一致(線上者覺得二等) 時區、語言、信用卡跨境反詐騙拒絕
關鍵 takeaway:付費複雜度真正的驅動因子是「觀眾的地理範圍」+「活動形式」這兩個正交軸。線下台灣活動最簡單(只需藍新);跨國 Hybrid 活動最複雜(藍新 + PayPal + SWIFT + 雙語收據都要)。設計平台時要先確認這兩軸,再決定金流 stack。
§8 EVENT STACKS

5 種活動類型 — 各自需要的 payment stack

同一份 netiCRM(或備案)可以服務 APAC AIA 各種活動,但每種活動只開啟需要的 stack:

活動類型 規模 付款方式 收據類型 範例
線上免費 webinar 50–300 人 無付款 不適用 AIA 雙月談
線下小型工作坊 20–50 人 單一藍新(信用卡 + ATM + 超商) 台灣 NPO 收據 WCAG 入門班
線下中型活動 50–150 人 藍新多票種 + 早鳥規則 台灣 NPO 收據 區域 a11y 論壇
Hybrid 中型 100–200 人 藍新(現場/國內線上)+ PayPal(國際線上) 雙語收據(中 + 英) (假設)APAC a11y day
大型國際年度 ★ 150–200+ 人 藍新 + PayPal + SWIFT(贊助) 雙語收據 + 英文 invoice ADALS 2026
採 netiCRM 最大價值:同一個平台支援所有活動類型 — 執行小組複製既有 event 設定即可建出簡化版本。不採 netiCRM(備案)的話,每種活動類型都要重新建立金流串接 + 收據邏輯 + 會員管理(省 NT$20K/年但每場活動都要重做)。
小型活動的 baseline 選項:Google Form + 銀行匯款 + 手工收據

對於 線上免費 webinar(僅收 Email 寄 Zoom 連結)和 線下小型工作坊(20-50 人、單一價位)兩類,如果不想搭任何平台,純手工流程是可行的:

  • 報名:Google Form(免費)
  • 付款:每位與會者 ATM 轉帳到玉山指定帳號,帳號後 5 碼當報名編號
  • 確認:祕書長 / 總務每日核對銀行對帳單,人工發確認 email + 寄手工收據

規模上限:大概 30 人以內。對 50+ 規模對帳人力成本太高;對 ADALS 規模(150-200 人)完全不可行。

適用情境:APAC AIA 未來辦小型 workshop 或 community 聚會、不想為單場活動建 netiCRM event 時的最簡 fallback。

§9 PATH DECISION

路徑判斷流程

三條路徑的完整 Price + Effort + Risk 比較見 §2 TL;DR。本節聚焦在「依條件判斷該走 A 還是 B」的決策流程。

判斷流程(Path A vs Path B)

  1. APAC AIA 是否預期持續舉辦活動 / 收會員費 / 收捐款?
    是 → 強烈傾向 Path A 否 → 評估 Path B
  2. NT$20K/年預算是否在執行小組可批的範圍?
    是 → Path A 可行 否 → 上 board 討論預算或退到 Path B
  3. AIA 主站是否已用 WordPress + Elementor?
    是 → Path B 候選成立(但需 vet plugin) 否 → 主推 Path A
  4. 諮詢 netiCRM 是否取得合理 NPO 條件?
    是 → 確認 Path A 否 → 退回 Path B 或重新諮詢
