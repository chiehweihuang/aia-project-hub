W0：全員並行準備（1 週）

以下 5 個人的任務同時進行，不是依序。全部完成 = W1 Christina 啟動。

Schneider (TBC) — 品牌 / 素材設計師

你的交付直接決定 Christina 能否開工。

  • Logo SVG 導出 — 中文版 + 英文版 × 淺底 + 深底，共 4 個 SVG 檔 + Favicon
  • 品牌色碼 — 主色、輔色、背景色（HEX 值）。需通過 WCAG 對比度，Bertram 會測
  • 字體確認 — 中文 + 英文字體名稱、粗細、層級規範

Claire (TBC) — 內容創作者

  • 內容盤點 + 標題層級規劃 — 首頁和活動頁的資訊架構文件（H1-H6 怎麼排）
  • 文案初稿 — 首頁 + 活動頁，中英雙語，每頁 300-600 字
  • 連結文字要有描述性（避免「了解更多」「按此」）

Becky (TBC) — 無障礙測試

  • 測試工具安裝 — axe DevTools、Colour Contrast Analyser、HeadingsMap、Landmarks 擴充
  • Mac：確認 VoiceOver 可用 | Windows：下載 NVDA
  • W3 起開始「邊建邊測」— Christina 建一頁你測一頁

Bertram — 專案經理

  • 從 Shared Drive 下載圖片 — 整理到 Web_Assets 資料夾 + 壓縮
  • 色碼對比度測試 — 拿到 Schneider 的色碼後立即用 CCA 測 WCAG 2.2 AA
  • 團隊照片選擇 — 每人 2 版選 1 版（依賴圖片下載完成）
  • 社群媒體帳號 URL 清單 — Facebook / LINE / YouTube / LinkedIn
  • 文案協助 — 與 Claire 共同撰寫首頁 + 活動頁文案

Christina — 網站設計師（Critical Path）

W0 你不需要做任何事。等素材到齊後（W1），你的工作開始：

  • W2-W3：Wireframe → Mockup（團隊 5 天內回饋）
  • W3-W5：Elementor 頁面建置 + Ally 外掛設定
  • W7：根據測試結果修正

團隊共同決定（W1 前）

  • Google Analytics 4 (GA4) 整合 — W3-W5

Kick-off Meeting Agenda（與 Christina）

啟動會議待確認事項。會議前請所有人先看過本頁內容。

1. 快速確認

  • WCAG 版本統一為 2.2 AA
  • 第三方費用清單 — Elementor 免費版還是 Pro？Ally 外掛授權費？字體授權？
  • Phase 2+ 頁面範圍 — 關於我們、團隊、加入我們、新聞/Blog、聯絡我們（跟章程一致？）

2. Staging 環境與測試流程

  • Staging URL — 建置期間我們怎麼看到進度？本地？還是有 staging 站？
  • 更新頻率 — 多久推一次更新到 staging？每週？每天？
  • 首頁先測？ — 首頁建好但活動頁還沒開始時，能先測首頁嗎？

3. 回饋與修改流程

  • 回報工具 — GitHub Issue？Google Sheet？還是 Christina 偏好的工具？
  • 無障礙問題 =/= 設計修改 — 無障礙是合規要求，不應計入「每里程碑 2 次修改」的額度。需要先確認這點。
  • 回饋時限 — 團隊 5 個工作日內回饋（第 3 天進度確認）
